# Uber配達進捗

Uber Eats 配達中に、目標件数に対して今の進捗が足りているかを確認するための小さな Web アプリです。

GitHub Pages で公開し、スマホのホーム画面に追加してアプリのように使う想定です。

## アプリ概要

- 目標件数、完了件数、残り稼働時間から、残り何件必要かを表示します。
- 終了上限を設定していない場合は、12時間制限を基準に進捗を表示します。
- 終了上限を設定した場合は、12時間制限の残り時間と終了上限までの時間を比較し、短い方を基準に表示します。
- 残り時間カウントを ON にすると、移動・停車にかかわらず残り時間を連続で減らせます。
- 残り時間カウントのON/OFFは、メイン画面下部の固定操作エリアから常に切り替えられます。
- 残り稼働時間は設定画面の時間・分プルダウンに加え、メイン画面の`−1分`・`＋1分`で細かく補正できます。長押し中は連続して変更できます。
- 小カードは3枚・4枚（2×2）・6枚（3×2）から表示枚数を選び、表示内容も変更できます。
- 画面右上の設定ボタンから、進捗条件と小カード設定をいつでも開けます。
- 小カードは設定から並べ替えモードを開始し、ドラッグして順番を変更できます。
- 小カードで同じ種類を選ぶと元のカードと入れ替わり、重複表示を防ぎます。
- 実績ペースを小カードに表示でき、目標ペースはメインカード内に固定表示します。
- 目標達成後は「今日のまとめ」に、完了件数・稼働時間・平均ペース・1時間あたり件数を表示します。
- 稼働計測の共通ボタンは、Uber時間ON中は休憩、Uber時間OFF中は他社稼働ON/OFFとして動作します。
- 他社稼働時間は総実稼働時間・実稼働率・履歴へ加算しますが、Uberの残り稼働時間やUber配達ペースには加算しません。

## 基本の使い方

1. 目標件数を設定します。
2. 完了件数を現在の配達完了数に合わせます。
3. 残り稼働時間を設定します。
4. 必要なら終了上限を設定します。
5. 配達が完了したら `＋1件`、取り消す場合は `−1件` を押します。タップでは1件、長押し中は連続して変更できます。

目標件数、完了件数、残り稼働時間、終了上限、カード表示は、画面右上の歯車ボタンから設定します。

## 入力項目の意味

### 目標件数

その日の目標配達件数です。

### 完了件数

すでに完了した配達件数です。

### 残り稼働時間

12時間制限のうち、あと何時間稼働できるかの目安です。

大きく合わせる場合は設定画面の時間・分プルダウンを使い、Uber Eats 配達アプリとの数分のずれはメイン画面の残り時間カウント内にある`−1分`・`＋1分`で微調整できます。タップでは1分、長押し中は連続して変更されます。残り時間カウント中に補正しても、カウントはそのまま継続します。

### 終了上限

「今日は何時までに終わりたいか」を入れる任意項目です。

- 終了上限なし: 12時間制限を基準に計算します。
- 終了上限あり: 12時間制限の残り時間と終了上限までの時間の短い方を有効残り時間として計算します。
- 現在時刻より前の時刻を入れた場合は、翌日の終了上限として扱います。例: 夜に `01:00` を設定すると `明日01:00上限` になります。

## 12時間制限との関係

このアプリは、12時間制限を固定の上限として扱います。

終了上限を設定していない場合は、従来通り12時間制限基準で表示します。

終了上限を設定した場合は、12時間制限の残り時間と終了上限までの時間を比較し、短い方を有効残り時間として計算します。

- 終了上限の方が短い場合は、終了上限基準で表示します。
- 12時間制限の方が短い場合は、12時間制限基準で表示します。
- 終了上限が12時間制限より後の場合、終了上限は補助情報として表示し、主計算には使いません。

現ペースで有効残り時間を超える見込みがある場合、終了予測には上限時刻を表示し、補足に `上限到達見込み` と表示します。

## 残り時間カウント

残り時間カウントは、Uber側の時計に手動で合わせるための補助機能です。

- Uber側の残り時間が進んでいる間は ON。
- Uber側の残り時間が止まっている時は OFF。
- ON中は移動・停車にかかわらず残り稼働時間が連続で減ります。
- 画面中央が`時間OFF`（Uber時計のカウント中）の時、稼働計測の共通ボタンは`休憩開始/終了`になります。
- 画面中央が`時間ON`（Uber時計の停止中）の時、同じボタンは`他社稼働ON/OFF`になります。
- 他社稼働中はUber残り時間を減らさず、総実稼働時間と実稼働率だけへ反映します。
- Uber時間ONへ切り替えると他社稼働は自動終了し、時間OFFへ切り替えると進行中の休憩は自動終了します。
- 案件の有無や端末の移動状態は自動判定せず、位置情報も使用しません。
- 内部では秒単位で計算し、画面には分単位で表示します。
- 手動で残り時間を変更すると、その値を基準に再計算します。
- Uber稼働時間は`12時間 − 現在の残り時間`と連携します。実稼働率は、Uber稼働時間と他社稼働時間の合計を、休憩除外後の経過時間で割って計算します。残り時間を補正した場合もUber稼働時間へ反映されます。
- `使い切り`には、現在から連続でカウントした場合に残り時間が0になる時刻を表示します。ON中は時刻が固定され、OFF中は時間の経過に合わせて後ろへ移動します。
- ON/OFFボタンは固定の`−1件`と`＋1件`の間に表示され、画面サイズにかかわらずメイン画面から操作できます。

## 遅れ時の復帰目安

余裕がある場合は、従来通りシングル・ダブルで受けられる時間の目安を表示します。

遅れている場合は、1件10分ペースを基準に、シングルならあと何件、ダブルならあと何組で予定へ戻れるかの目安を表示します。ダブル1組は20分（平均10分/件）として扱います。

目標ペースが10分/件以下の場合は、10分/件では遅れを回復できない旨を表示します。

遅れは表示上18分まで橙、19分から赤で表示します。

## ホーム画面に追加して使う

スマホのブラウザで GitHub Pages の公開URLを開き、iPhone なら Safari の共有メニューから「ホーム画面に追加」して使う想定です。

PWA表示、スマホ表示、小さめiPhone向けの表示調整を入れています。

## 注意点

- 表示される終了予測や必要ペースは、スマホの現在時刻を使った目安です。
- localStorage に入力値、終了上限、カード設定、残り時間カウント状態を保存します。
- 端末やブラウザの localStorage を削除すると、保存内容も消えます。
- リセットすると、完了件数、残り稼働時間、終了上限、当日の稼働計測が初期状態に戻ります。記録済みの稼働がある場合は、リセット前に履歴へ保存できます。
